# rules_cn Profile

这个 profile 面向当前项目默认业务场景：`培训管理 + 劳动法` 双知识域。

## 适用范围

- 培训机构规章制度、考勤、请假、宿舍、奖惩、退学等管理制度问答
- 劳动法、劳动合同法、试用期、工资、辞退、社保、经济补偿等法律问答

## 配置特点

- 自动路由会优先区分 `training_management` 与 `labor_law`
- 知识库创建时的业务域选项与默认域由 `knowledge_base_domains.json` 统一提供，不再由前端本地写死
- FAQ 高频问法短路会覆盖“请假制度 / 考勤制度 / 劳动法概览”这类短问
- 文档切分会对 `plain text / OCR text` 使用更细的中文子句切分 profile
- 检索规则会保留制度类文档中与扣分、处罚、退学标准相关的关键片段
- overview 识别会通过 `overview_rules.json` 识别“请假一般怎么处理”“课堂违纪一般会怎么处理”这类总括问题
- query alias 会把劳动法和培训管理中的口语问法扩展成更贴近原文的检索词
- answer guard 会对“实习期间”这类限定场景进行保守回答保护
- query rewriter prompt 会使用当前场景的关键词示例和规则改写

## 维护建议

- 如果你只是在当前业务场景内微调效果，优先修改本目录下文件
- 如果你在做新的业务类型，不要继续往本目录塞规则，应该新建一个 profile
