# ReguRAG 术语对照表

这份表只做项目内常见说法的对照，不改 `项目进度.md` 里的原文写法。

## 回归和验证

| 原词 | 中文直译 | 在本项目里的实际意思 |
| --- | --- | --- |
| smoke | 冒烟测试 | 最小验证；先确认链路活着、不会一跑就炸 |
| retrieval smoke | 检索冒烟 | 小样本快速验证 `ingest -> retrieve -> rerank -> citation` |
| shadow smoke | 影子链路冒烟 | 小样本快速验证 shadow compare 会不会报错、会不会明显漂移 |
| mixed smoke | 混合恢复最小验证 | 把 `retry + stale reclaim` 混在一起做一次小样本恢复验证 |
| full regression | 全量回归 | 用完整样本认真检查是否退化 |
| staged regression | 分阶段回归 | 把 `retrieval / live_eval / shadow` 拆开跑，支持复用已有结果 |
| rollout gate | 灰度准入门槛 | 判断新后端能不能进入下一阶段的硬条件 |
| ready_for_rollout | 可灰度 | 所有准入门槛都满足时的最终结论 |

## 任务队列和恢复

| 原词 | 中文直译 | 在本项目里的实际意思 |
| --- | --- | --- |
| retry | 重试 | 任务失败后回到 `pending`，再被重新执行 |
| stale | 失活中的旧任务 | 表面还在 `running`，但原 worker 已经失活 |
| reclaim | 接管 | 新 worker 把 stale 任务重新认领过来继续跑 |
| stale reclaim | 失活任务接管 | 针对 stale 任务的恢复机制 |
| mixed recovery | 混合恢复 | 同时验证 `retry` 和 `stale reclaim` 两种恢复路径 |
| one-off worker | 一次性 worker | 为某次验证临时拉起、跑完就停的 worker |

## 检索链和评测

| 原词 | 中文直译 | 在本项目里的实际意思 |
| --- | --- | --- |
| vector store | 向量存储 | 主链 dense 检索后端，目前可切 `chroma / milvus` |
| shadow | 影子链路 | 不影响主结果的旁路对照链，用来比较新旧实现 |
| shadow compare | 影子对比 | 同一问题同时跑主链和影子链，再比较结果 |
| live eval | 真实在线评测 | 用真实模型接口跑完整 `retrieve -> rerank -> generate` |
| retrieval-only | 仅检索评测 | 不看生成答案，只比较检索、最终上下文、引用 |
| drift | 漂移 | 新后端和基线后端的结果不一致 |
| parity | 对齐 | 新后端和基线后端能力一致，没有明显退化 |

## 当前建议说法

- `smoke`：后续可以直接说“最小验证”
- `mixed smoke`：后续可以直接说“混合恢复最小验证”
- `shadow smoke`：后续可以直接说“影子链路最小验证”
- `full regression`：后续可以直接说“全量回归”
