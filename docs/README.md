# docs 目录说明

`docs/` 根目录保留当前仍在使用、与现有项目状态一致的说明文档。历史阶段的规划稿、联调说明、旧版构建文档和一次性迁移指南放入 `docs/archive/`。

## 当前有效文档

- `backend-uv-migration-guide.md`：非 Docker 本地后端开发或新设备 uv 环境初始化说明。
- `知识库文档接入规范.md`：知识库文档上传与预处理规范。
- `术语对照表.md`：当前项目核心术语和命名对照。
- `后端主链路阅读地图.md`：后端主链路阅读路径。
- `后端模块职责一览.md`：后端模块边界和职责说明。
- `后端代码阅读顺序 Checklist.md`：后端代码阅读顺序。
- `全栈代码阅读入口.md`：从前端页面到后端服务的全栈阅读入口。
- `概览型分情形问题评测样本设计.md`：overview 类评测样本设计说明。
- `主链Milvus灰度切换方案.md`：Milvus 主链灰度切换、回滚和验证方案。

## 当前归档文档

- `archive/backend-build-guide.md`
- `archive/frontend-api-handoff.md`
- `archive/frontend-build-guide.md`

归档文档只保留项目演化过程，不应默认视为当前实现状态。

## 使用约定

- 需要指导当前开发、运行、迁移的内容，放在 `docs/` 根目录。
- 已经过时但仍有历史参考价值的内容，移入 `docs/archive/`。
- 如果某份文档已经失去参考价值，可以直接删除，不必长期保留。
