# 全栈代码阅读入口

这份文档用于快速定位代码，不代替更细的后端阅读地图。

## 1. 先从三条用户路径读

### 1.1 问答路径

前端入口：

- `frontend/src/views/ChatView.vue`
- `frontend/src/stores/chat.ts`
- `frontend/src/api/chat.ts`

后端入口：

- `backend/app/api/endpoints/chat.py`
- `backend/app/services/rag_service.py`
- `backend/app/rag/pipeline.py`

阅读顺序：

1. `ChatView.vue` 看页面如何发送问题、展示流式回答、打开引用和 debug。
2. `stores/chat.ts` 看 SSE 的 `start/token/end` 如何变成本地消息状态。
3. `api/chat.ts` 看前端如何解析后端 SSE。
4. `endpoints/chat.py` 看 HTTP/SSE 响应如何封装。
5. `RAGService` 看知识库路由、短路、answer guard 和持久化。
6. `RAGPipeline` 看查询改写、检索、MMR、rerank、上下文和 LLM 生成。

### 1.2 文档入库路径

前端入口：

- `frontend/src/views/DocumentsView.vue`
- `frontend/src/stores/task.ts`
- `frontend/src/api/documents.ts`
- `frontend/src/api/tasks.ts`

后端入口：

- `backend/app/api/endpoints/documents.py`
- `backend/app/api/endpoints/knowledge_bases.py`
- `backend/app/services/ingest_service.py`
- `backend/app/workers/task_worker.py`

阅读顺序：

1. `DocumentsView.vue` 看上传、导入、重建、删除按钮如何防重复提交。
2. `stores/task.ts` 看前端如何恢复和轮询 pending/running 任务。
3. `documents.py` / `knowledge_bases.py` 看 HTTP 请求如何创建后台任务。
4. `IngestService` 看文件格式预处理、全量/部分 rebuild 和任务事件。
5. `TaskWorker` 看任务认领、失败重试和 worker 常驻运行。

### 1.3 任务观测路径

前端入口：

- `frontend/src/views/TaskMonitorView.vue`
- `frontend/src/components/tasks/`
- `frontend/src/api/tasks.ts`

后端入口：

- `backend/app/api/endpoints/tasks.py`
- `backend/app/services/ingest_service.py::TaskService`
- `backend/app/repositories/metadata_repository.py`

阅读顺序：

1. `TaskMonitorView.vue` 看 overview、alerts、trends、task events 如何并行加载。
2. `components/tasks/` 看每个观测模块如何展示数据。
3. `tasks.py` 看接口如何暴露任务列表、告警、趋势和事件时间线。
4. `TaskService` 看监控指标如何组织。
5. `MetadataRepository` 看 SQL 层如何统计 stale running、active workers 和近期失败。

### 1.4 测试和评测路径

测试入口：

- `backend/tests/conftest.py`
- `backend/tests/test_rag_end_to_end_integration.py`
- `backend/tests/test_task_worker_sql_recovery_integration.py`
- `backend/tests/test_vector_store_regression_eval.py`

评测入口：

- `backend/scripts/run_shadow_retrieval_regression.py`
- `backend/app/evals/shadow_compare.py`
- `backend/app/evals/shadow_regression.py`
- `backend/app/evals/vector_store_regression.py`
- `backend/app/evals/eval_report_compare.py`

阅读顺序：

1. `conftest.py` 看 API 单测如何绕开真实启动依赖。
2. `test_rag_end_to_end_integration.py` 看主链路用哪些确定性替身固定行为。
3. `test_task_worker_sql_recovery_integration.py` 看 SQL 队列如何验证重试、租约过期和部分文档失败。
4. `run_shadow_retrieval_regression.py` 看 Chroma/Milvus shadow 回归如何批量跑样本。
5. `shadow_compare.py` 看 legacy pipeline 和 shadow graph 如何做逐样本对比。
6. `vector_store_regression.py` 看检索、真实问答和 shadow graph 指标如何合成上线门禁。

常用脚本分组：

- 问答评测：`backend/scripts/eval_rag.py`、`backend/scripts/compare_eval_reports.py`
- 流式性能：`backend/scripts/benchmark_chat_stream.py`、`backend/scripts/compare_stream_benchmark.py`
- 分块/稀疏检索 A/B：`backend/scripts/compare_split_profiles.py`
- Shadow graph：`backend/scripts/compare_shadow_graph.py`、`backend/scripts/run_shadow_retrieval_regression.py`
- 向量库灰度：`backend/scripts/compare_real_vector_store_retrieval.py`、`backend/scripts/compare_real_vector_store_eval.py`、`backend/scripts/run_vector_store_regression.py`

- 意图分类：`backend/scripts/validate_intent_dataset.py`、`backend/scripts/train_intent_classifier.py`、`backend/scripts/eval_intent_classifier.py`

## 2. 后端分层记法

- `api/endpoints/`：只做 HTTP schema 转换和 SSE 编码。
- `services/`：业务编排层，负责路由、任务、持久化和保护逻辑。
- `rag/`：核心 RAG 能力，负责检索、重排、向量库和 LLM 调用。
- `repositories/`：数据库读写层，屏蔽 MySQL/PostgreSQL 细节。
- `workers/`：后台任务执行进程。
- `workflows/rag/`：shadow graph，对照 legacy pipeline 的迁移实现。
- `tests/`：覆盖 API、服务、RAG、worker、向量库灰度和评测汇总。
- `app/evals/` + `scripts/`：评测数据加载、报告比较和灰度回归脚本。
