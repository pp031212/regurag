# 后端模块职责一览

这份文档只回答一个问题：

> 每个核心模块到底负责什么，不负责什么。

如果你拉到一段代码先分不清它属于哪层，就先看这张表。

---

## 1. `RAGService`

位置：

- [rag_service.py](/D:/System/Desktop/regurag/backend/app/services/rag_service.py:684)

负责：

- 请求总入口
- 知识库路由
- 会话解析
- 轻意图 / FAQ / off-topic 短路
- cross-domain guard
- 调用 `RAGPipeline`
- answer guard
- 持久化 message / `message_context`
- shadow compare 挂载

不负责：

- 真正的 dense / sparse 检索实现
- rerank 细节
- final context 选择细节

一句话：

> `RAGService` 是服务层编排器，管外围治理和落库，不管核心检索算法细节。

---

## 2. `RAGPipeline`

位置：

- [pipeline.py](/D:/System/Desktop/regurag/backend/app/rag/pipeline.py:438)

负责：

- 核心问答流水线
- query 输入准备
- 调 `Retriever`
- MMR
- parent 聚合
- rerank
- final context 构建
- 调 LLM 生成答案
- 构建 citations / debug

不负责：

- 知识库自动路由
- FAQ / 轻意图短路
- 最终消息持久化

一句话：

> `RAGPipeline` 是核心问答流水线，不处理服务层外围治理。

---

## 3. `Retriever`

位置：

- [hybrid_retriever.py](/D:/System/Desktop/regurag/backend/app/rag/retrievers/hybrid_retriever.py:14)

负责：

- dense 检索
- sparse 检索
- merge / dedupe / sort
- 输出统一 retrieval 结果

不负责：

- 会话历史处理
- rerank
- 最终答案生成

一句话：

> `Retriever` 只负责“把候选文档召回出来并整理好”。

---

## 4. `VectorStore`

位置：

- [vector_store.py](/D:/System/Desktop/regurag/backend/app/rag/vector_store.py:13)

负责：

- chunk embedding 写入
- Chroma collection 管理
- dense search
- 当前 sparse provider 所需索引维护
- 文档删除 / collection reset

不负责：

- retrieval 策略编排
- MMR / rerank / final context

一句话：

> `VectorStore` 是索引和底层检索存取层，不是完整 Retriever。

---

## 5. `RetrievalPolicy`

位置：

- [policy.py](/D:/System/Desktop/regurag/backend/app/rag/retrievers/policy.py:7)

负责：

- `dense_top_k`
- `sparse_top_k`
- `sparse_min_hits`
- `enable_sparse`

不负责：

- rerank 开关
- answer style
- 服务层路由策略

一句话：

> `RetrievalPolicy` 决定“这次检索怎么跑”。

## 6. `QueryPolicy`

位置：

- [query_policy.py](/D:/System/Desktop/regurag/backend/app/rag/query_policy.py:9)

负责：

## 7. `shadow graph`

位置：

- [graph.py](/D:/System/Desktop/regurag/backend/app/workflows/rag/graph.py:13)

## 8. `ConversationContextResolver`

位置：

- [conversation_context_resolver.py](/D:/System/Desktop/regurag/backend/app/services/conversation_context_resolver.py:15)

负责：

---

## 10. `KnowledgeBaseRouter`

位置：

- [knowledge_base_router.py](/D:/System/Desktop/regurag/backend/app/services/knowledge_base_router.py:217)

负责：

- 在多知识库之间选更合适的目标知识库

不负责：

一句话：

> 它决定“去哪个库问”，不是“在库里怎么问”。

---

## 11. 最简分层记忆法

如果你只想记一个版本，就记下面这组：

- `RAGService`：外围治理层
- `RAGPipeline`：核心流水线层
- `Retriever`：候选召回层
- `VectorStore`：索引存取层
- `Policy`：策略配置层
- `shadow graph`：核心对照层

也就是：

> 服务层管入口和收尾，pipeline 管主流程，retriever 管召回，vector store 管索引，policy 管策略，shadow 管对照。
